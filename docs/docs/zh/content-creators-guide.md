# 目标

本存储库的目标是为 AI Tour 26 活动提供一致的客户体验并提高研讨会文档的质量。通过遵循建议的结构和最佳实践，您可以确保您的内容清晰、有吸引力且易于所有参与者理解。

欢迎使用 AI Tour 26 文档工具包！本指南将帮助您为您的受众创建有效且引人入胜的内容。

## 实验室的建议结构

本存储库为您的文档提供了推荐的结构。对于实验室指南（请参阅 `lab-1-open-workshop.md` 作为参考），每个实验室应包括：

- **您将学到什么：** 简要概述参与者将获得的关键技能或概念。

- **介绍：** 设置背景并解释实验室的目的。

- **实验室练习：** 完成实验室任务的分步说明。

遵循此格式有助于确保所有研讨会材料的一致性和清晰性。

## 了解您的受众

- **开发人员或数据科学家：** 专注于实际示例、代码片段和动手实验室。

- **技术决策者：** 突出架构、业务价值和战略见解。

    !!! 提示
        针对 **数据科学家** 的内容，Jupyter Notebooks 是呈现交互式代码、数据分析和可视化的绝佳方式。

## KISS 原则

- **保持简单：** 避免术语和复杂的解释。假设最少的先验知识。

- **清晰：** 使用简洁的语言和分步说明。

## 开始使用此存储库

- **克隆此存储库：**

  ```sh
  git clone <your-repo-url>
  ```

- **复制文档：**
  将此存储库中的 `docs` 文件夹的内容复制到您的 AI Tour 26 存储库中的 `docs` 文件夹。

## 开始使用 MkDocs

- **安装依赖项：**
  MkDocs 的 `requirements.txt` 文件位于 `docs` 文件夹中。使用以下命令安装依赖项：

    ```sh
    pip install -r docs/requirements.txt
    ```
