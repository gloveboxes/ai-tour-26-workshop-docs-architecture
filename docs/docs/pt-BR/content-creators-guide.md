## Objetivo

O objetivo deste repositório é fornecer uma experiência consistente ao cliente e melhorar a qualidade da documentação do workshop para os eventos do AI Tour 26. Seguindo a estrutura sugerida e as melhores práticas, você pode garantir que seu conteúdo seja claro, envolvente e fácil de seguir para todos os participantes.

Bem-vindo ao kit de ferramentas de documentação do AI Tour 26! Este guia ajudará você a criar conteúdo eficaz e envolvente para seu público.

## Estrutura Sugerida para Laboratórios

Este repositório fornece uma estrutura recomendada para sua documentação. Para guias de laboratório (veja `lab-1-open-workshop.md` como referência), cada laboratório deve incluir:

- **O que Você Aprenderá:** Resuma as principais habilidades ou conceitos que os participantes adquirirão.
- **Introdução:** Contextualize e explique o propósito do laboratório.
- **Exercício de Laboratório:** Instruções passo a passo para concluir as tarefas do laboratório.

Seguir este formato ajuda a garantir consistência e clareza em todos os materiais do workshop.

## Entenda Seu Público

- **Desenvolvedores ou Cientistas de Dados:** Foque em exemplos práticos, trechos de código e laboratórios práticos.
- **Tomadores de Decisão Técnicos:** Destaque arquitetura, valor para os negócios e insights estratégicos.

    !!! dica
        Para conteúdo voltado a **Cientistas de Dados**, Jupyter Notebooks são uma ótima maneira de apresentar código interativo, análise de dados e visualizações.

## Princípio KISS

- **Mantenha Simples:** Evite jargões e explicações complexas. Presuma conhecimento prévio mínimo.
- **Seja Claro:** Use linguagem concisa e instruções passo a passo.

## Comece com Este Repositório

- **Clone este repositório:**

  ```sh
  git clone <seu-url-repo>
  ```

- **Copie os documentos:**
  Copie o conteúdo da pasta `docs` deste repositório para a pasta `docs` no seu repositório do AI Tour 26.

## Começando com o MkDocs

- **Instale os requisitos:**
  O arquivo `requirements.txt` para MkDocs está na pasta `docs`. Instale as dependências com:

    ```sh
    pip install -r docs/requirements.txt
    ```
