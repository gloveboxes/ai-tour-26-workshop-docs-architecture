## Layunin

Ang layunin ng repositoryong ito ay magbigay ng pare-parehong karanasan para sa mga customer at mapabuti ang kalidad ng dokumentasyon ng workshop para sa mga kaganapan ng AI Tour 26. Sa pamamagitan ng pagsunod sa iminungkahing istruktura at mga pinakamahusay na kasanayan, masisiguro mong malinaw, nakakaengganyo, at madaling sundan ang iyong nilalaman para sa lahat ng kalahok.

Maligayang pagdating sa toolkit ng dokumentasyon ng AI Tour 26! Ang gabay na ito ay tutulong sa iyo na lumikha ng epektibo at nakakaengganyong nilalaman para sa iyong audience.

## Iminungkahing Istruktura para sa Mga Lab

Ang repositoryong ito ay nagbibigay ng inirerekomendang istruktura para sa iyong dokumentasyon. Para sa mga gabay sa lab (tingnan ang `lab-1-open-workshop.md` bilang sanggunian), ang bawat lab ay dapat maglaman ng:

- **Ano ang Matututuhan Mo:** Maikling outline ng mga pangunahing kasanayan o konseptong makakamit ng mga kalahok.
- **Panimula:** Itakda ang konteksto at ipaliwanag ang layunin ng lab.
- **Lab Exercise:** Hakbang-hakbang na mga tagubilin para sa pagsasagawa ng mga gawain sa lab.

Ang pagsunod sa format na ito ay nakakatulong upang masiguro ang pare-pareho at malinaw na nilalaman sa lahat ng materyales ng workshop.

## Unawain ang Iyong Audience

- **Mga Developer o Data Scientist:** Mag-focus sa mga praktikal na halimbawa, code snippets, at hands-on labs.
- **Mga Technical Decision Makers:** I-highlight ang arkitektura, halaga ng negosyo, at mga estratehikong insight.

    !!! tip
        Para sa nilalamang nakatuon sa **Data Scientists**, ang Jupyter Notebooks ay isang mahusay na paraan upang magpakita ng interactive na code, pagsusuri ng data, at mga visualization.

## Prinsipyo ng KISS

- **Panatilihing Simple:** Iwasan ang jargon at komplikadong paliwanag. I-assume ang minimal na kaalaman.
- **Maging Malinaw:** Gumamit ng maikli at malinaw na wika at hakbang-hakbang na mga tagubilin.

## Magsimula sa Repositoryong Ito

- **I-clone ang repositoryong ito:**

  ```sh
  git clone <your-repo-url>
  ```

- **Kopyahin ang mga docs:**
  Kopyahin ang nilalaman ng folder na `docs` mula sa repositoryong ito papunta sa folder na `docs` sa iyong AI Tour 26 repository.

## Pagsisimula sa MkDocs

- **I-install ang mga kinakailangan:**
  Ang `requirements.txt` file para sa MkDocs ay nasa folder na `docs`. I-install ang mga dependencies gamit ang:

    ```sh
    pip install -r docs/requirements.txt
    ```
