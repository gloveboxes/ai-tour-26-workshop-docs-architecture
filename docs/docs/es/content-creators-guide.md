## Objetivo

El objetivo de este repositorio es proporcionar una experiencia consistente para el cliente y mejorar la calidad de la documentación del taller para los eventos de AI Tour 26. Siguiendo la estructura sugerida y las mejores prácticas, puede asegurarse de que su contenido sea claro, atractivo y fácil de seguir para todos los participantes.

¡Bienvenido al kit de herramientas de documentación de AI Tour 26! Esta guía le ayudará a crear contenido efectivo y atractivo para su audiencia.

## Estructura sugerida para laboratorios

Este repositorio proporciona una estructura recomendada para su documentación. Para las guías de laboratorio (vea `lab-1-open-workshop.md` como referencia), cada laboratorio debe incluir:

- **Lo que aprenderá:** Resuma brevemente las habilidades o conceptos clave que los participantes adquirirán.
- **Introducción:** Establezca el contexto y explique el propósito del laboratorio.
- **Ejercicio de laboratorio:** Instrucciones paso a paso para completar las tareas del laboratorio.

Seguir este formato ayuda a garantizar la consistencia y claridad en todos los materiales del taller.

## Comprenda a su audiencia

- **Desarrolladores o científicos de datos:** Concéntrese en ejemplos prácticos, fragmentos de código y laboratorios prácticos.
- **Tomadores de decisiones técnicos:** Destaque la arquitectura, el valor empresarial y las ideas estratégicas.

    !!! consejo
        Para contenido dirigido a **científicos de datos**, los Jupyter Notebooks son una excelente manera de presentar código interactivo, análisis de datos y visualizaciones.

## Principio KISS

- **Manténgalo simple:** Evite jerga y explicaciones complejas. Suponga un conocimiento previo mínimo.
- **Sea claro:** Use un lenguaje conciso e instrucciones paso a paso.

## Comience con este repositorio

- **Clone este repositorio:**

  ```sh
  git clone <su-url-repo>
  ```

- **Copie los documentos:**
  Copie el contenido de la carpeta `docs` de este repositorio a la carpeta `docs` en su repositorio de AI Tour 26.

## Comenzando con MkDocs

- **Instale los requisitos:**
  El archivo `requirements.txt` para MkDocs está en la carpeta `docs`. Instale las dependencias con:

    ```sh
    pip install -r docs/requirements.txt
    ```
