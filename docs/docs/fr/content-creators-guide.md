## Objectif

L'objectif de ce référentiel est de fournir une expérience client cohérente et d'améliorer la qualité de la documentation des ateliers pour les événements AI Tour 26. En suivant la structure suggérée et les meilleures pratiques, vous pouvez garantir que votre contenu est clair, engageant et facile à suivre pour tous les participants.

Bienvenue dans la boîte à outils de documentation AI Tour 26 ! Ce guide vous aidera à créer un contenu efficace et engageant pour votre public.

## Structure suggérée pour les laboratoires

Ce référentiel fournit une structure recommandée pour votre documentation. Pour les guides de laboratoire (voir `lab-1-open-workshop.md` comme référence), chaque laboratoire doit inclure :

- **Ce que vous apprendrez :** Résumez brièvement les compétences ou concepts clés que les participants acquerront.
- **Introduction :** Mettez en contexte et expliquez l'objectif du laboratoire.
- **Exercice de laboratoire :** Instructions étape par étape pour réaliser les tâches du laboratoire.

Suivre ce format aide à garantir la cohérence et la clarté de tous les supports d'atelier.

## Comprendre votre public

- **Développeurs ou Data Scientists :** Concentrez-vous sur des exemples pratiques, des extraits de code et des laboratoires pratiques.
- **Décideurs techniques :** Mettez en avant l'architecture, la valeur commerciale et les insights stratégiques.

    !!! conseil
        Pour le contenu destiné aux **Data Scientists**, les Jupyter Notebooks sont un excellent moyen de présenter du code interactif, des analyses de données et des visualisations.

## Principe KISS

- **Keep It Simple (Restez simple) :** Évitez le jargon et les explications complexes. Supposez des connaissances préalables minimales.
- **Soyez clair :** Utilisez un langage concis et des instructions étape par étape.

## Commencez avec ce référentiel

- **Clonez ce référentiel :**

  ```sh
  git clone <votre-url-repo>
  ```

- **Copiez les documents :**
  Copiez le contenu du dossier `docs` de ce référentiel dans le dossier `docs` de votre référentiel AI Tour 26.

## Commencer avec MkDocs

- **Installez les dépendances :**
  Le fichier `requirements.txt` pour MkDocs se trouve dans le dossier `docs`. Installez les dépendances avec :

    ```sh
    pip install -r docs/requirements.txt
    ```
